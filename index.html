RunLoop = Main Run Loop + Update Cycle

为了便于理解,可以将整个App的RunLoop看做是由以上两个阶段组成.
App在Main Run Loop Update Cycle阶段, 如果需要重新绘制, 布局等等.会给对应视图做好标记,
等到Update Cycle时会真正开始对视图进行修改.整篇文章也围绕着以上理解展开.

前言

  • controller本身是一个很普通的类, 析构和构造与任意新建的类没有什么不同, 而通常开发者所说的生命周期,并不是指controller类本身的生命周期,而是指代其维护的视图层次的不同执行阶段.
  • controller有一个view属性, 而这个view就是所维护的视图层次的根视图. 其中, view属性采用了懒加载的方式, 即第一次访问该属性时才会加载或者创建它.

RxSwift基础整理

发表于 2017-12-13 | 分类于 iOS ， RxSwift

初学rxswift，记录一些相关知识点，加深印象，如果有错误地方，麻烦指教，谢谢。

RxSwift 背景相关

Rx即响应式编程，是建立在观察者设计模式上的异步响应方式.
iOS开发中,说到异步API可能会想到通知,委托,GCD, 闭包.不过多种方式结合,有时往往导致写出清晰明了的异步代码变得比较困难.Rx很好的解决了这个问题,而且Rx家族涵盖了很多流行语言,因此也会极大降低学习其他平台的成本.

RxSwift只是语言层面上实现了Rx API,而其对Cocoa和UIKit相关类是一无所知的,RxCocoa为开发者搭建了两者间的桥梁.
