Layer Animations - Part I

发表于 2017-12-13 | 分类于 iOS ， Animation | 次

概念性解释

  1. View 与 Layer对比

Views

  • 复杂的视图层级，自动布局
  • 用户交互
  • 通常有定制逻辑或者定制绘制，执行在主线程，通过CPU执行
  • 比较灵活，更加强大，有许多可用的类

Layers

  • 层级简单，布局和绘制迅速
  • 没有响应链
  • 没有定制逻辑，直接在GPU绘制
  • 没有view那么灵活

Core Animation 维护了两个平行 layer 层次结构： model layer tree（模型层树） 和 presentation layer tree（表示层树）。前者中的 layers 反映了我们能直接看到的 layers 的状态，而后者的 layers 则是动画正在表现的值的近似

render tree(private) 在apple的render server进程中，是真正处理动画的地方。而且线程的优先级也比我们主线程优先级高。所以有时候即使我们的App主线程busy，依然不会影响到手机屏幕的绘制工作

presentation layer tree是layer在屏幕中的真实位置，即动画修改的都是presentation layer tree,一旦动画被移除，presentation layer tree会重新变为model layer tree的值.

注：通常情况下，view animations已经可满足动画需求，所以每次应该优先考虑view animations，后面追求性能或者其他更灵活的需求时，再结合layer动画

Animatable properties(动画属性)

  1. 位置大小 (bounds, position, transform)
  2. 边界（borderColor, borderWidth, cornerRadius）
  3. 阴影(shadowOffset, shadowOpacity, shadowPath, shadowRadius)
  4. 内容（contents, mask, opacity）

CAAnimation简介

  1. 动画对象父类。抽象类。
属性 说明
duration 总时长
repeatCount 重复次数
repeatDuration 重复时间
autoreverses 自动逆转动画
speed 动画速度
removedOnCompletion 动画执行完毕，是否从图层上移除，设置位NO需要搭配kCAFillModeForwards
fillMode 控制动画开始和结束后，layer的行为
beginTime 动画开始时间，多用于延迟CACurrentMediaTime()
timingFunction 速度控制
delegate 动画代理
  • fillMode
选项 说明
kCAFillModeRemoved 动画开始时，layer图层才变为动画设置的初始状态，动画结束后，layer回到之前的状态
kCAFillModeBackwards 忽略动画开始的时机，当动画加入layer，layer的presentation layer tree的值就变为动画的初始状态值
kCAFillModeForwards 动画结束后，presentation layer tree会一直保存动画结束的状态值
kCAFillModeBoth 结合以上两者
  • timingFunction
选项 说明
kCAMediaTimingFunctionLinear 匀速
kCAMediaTimingFunctionEaseIn 汽车加速（先慢后快）
kCAMediaTimingFunctionEaseOut 汽车停车（减速刹停）
kCAMediaTimingFunctionEaseInEaseOut 结合以上两者
