Layer Animations - Part Ⅱ

发表于 2017-12-13 | 分类于 iOS ， Animation | 次

委托(delegates)

之前提到的UIKit animations就像一支开弓的弓箭，一旦创建执行，那么就没有机会暂停或者停止它。而Core Animation, 可以通过委托回调轻易的控制动画。

键值编码兼容(key-value coding compliance)

假如一个界面上有多个动画内容，如果在委托回调中区分不同的动画呢？

CAAnimation具键值编码兼容的特性，你可以讲他们看作是字典，可以在运行时向其添加新的属性。

后续我们就可以通过key和对应的value来标识动画，以及对layer进行后续的操作。

CAAnimation子类介绍

CAPropertyAnimation(属性动画)

抽象类，本身不能对layer进行动画操作，需使用其子类CABasicAnimation,CAKeyframeAnimation

属性 说明
keyPath
CABasicAnimation(基本动画)
属性 说明
fromValue keyPath属性初始值
toValue keyPath属性最终值
byValue keyPath属性改变量
CAAnimationGroup（动画组）
属性 说明
animations 动画对象数组
CASpringAnimation(弹簧动画)
属性 说明
damping 阻尼
mass 物体重量
stiffness 刚度，抗形变能力
initial velocity 初速度

CASpringAnimation是CABasicAnimation的子类

CAKeyframeAnimation(关键帧动画)

CAKeyframeAnimation是CAPropertyAnimation的子类
通俗理解，可以自由控制动画“每帧”的展示，整个动画执行过程都必须被覆盖到，不能断片。(CABasicAnimation就相当于只有首尾两帧的CAKeyframeAnimation)

属性 说明
values 所有帧的集合数组，在动画时长内依次显示
path 设置了path，那么values将被忽略
keyTimes 关键帧对应的时间点，一一对应，如果没有设置，则时间平分

struct是swift的一等公民，与class的特性有许多不同。Core Animation是oc framework，且由c组成。所以针对Strut的动画api，需要一些特殊处理。
